# Security Policy

## Supported Versions

Currently, the following versions of **Adaptive Cachy Beauty** are actively supported with security updates. We highly recommend using the latest `master` branch or the latest beta release.

| Version | Supported          |
| ------- | ------------------ |
| v0.5.x  | :white_check_mark: |
| < v0.5  | :x:                |

## Reporting a Vulnerability

Security is a top priority, especially for a tool that handles system-level configuration files (like terminal configs, Zsh profiles, and KDE DBus signals).

If you discover a vulnerability (e.g., unintended code execution via malicious wallpaper metadata or path traversal in configuration parsing), **do NOT open a public Issue.**

Instead, please responsibly disclose it by sending an email directly to the maintainer:
**Email:** `dimarch0x` (via standard GitHub profile channels) or send a direct message.
Alternatively, use the GitHub **Security Advisories** feature to privately report a vulnerability in this repository.

### What to include in your report

- A detailed description of the vulnerability.
- Steps to reproduce it.
- Possible impact if exploited.

**Response Time:**
We aim to acknowledge receipt of vulnerability reports within 48 hours and provide a patch/update as swiftly as possible before public disclosure.
